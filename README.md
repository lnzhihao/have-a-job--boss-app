# BOSS 找工作 Agent

帮你在 BOSS 直聘上**半自动找工作**的本地小工具：从职位列表里按你的标准筛选 → 用 AI 给每个岗位做 10 维度匹配打分 → 自动「立即沟通」打招呼 → HR 一回复就自动发作品集链接和简历 → 后台看板实时显示投了多少、匹配度、公司、薪资、地区。

> 为「海外新媒体运营 / 海外营销」方向定制，关键词、薪资、区域都可在界面里改。

---

## 它是怎么工作的（重要）

BOSS 直聘对「程序自己启动的浏览器」做了反爬——会把职位列表整页返回**空白**。所以本工具**不自己开浏览器**，而是**连接你自己正常打开、已登录的 Chrome**（通过调试端口）。在你的真实会话里操作，BOSS 不屏蔽，列表正常加载。

```
你的真实 Chrome（调试模式, 已登录 BOSS）
        ▲  Playwright connectOverCDP
        │
   Node 后台 (server.js + engine.js)  ──SSE──►  浏览器里的网页控制台 (localhost:8848)
        │
   DeepSeek / Claude API（评分 & 求职建议）
```

- **流程一（打招呼）**：读职位列表 → 关键词/区域/薪资/HR活跃度筛选 → AI 10 维评分(≥阈值才投) → 拟人「立即沟通」(用你 BOSS 设置的默认招呼语，不自拟话术)。
- **流程二（发作品集/简历）**：轮询消息页，HR 一回复就先发**作品集链接**、再点 BOSS 原生「发简历」发**简历附件**，然后通知你接手（海投模式：有人回复即发，不等第二句）。
- **看板**：已打招呼 / HR回复 / 已发简历 / 已跳过，外加每个岗位的匹配分、公司、薪资、地区（日志 + 建议页）。

## 安装

需要 Node.js 18+ 和本机 Google Chrome。

```bash
npm install
npx playwright install chromium   # 装一次（仅做依赖完整性，实际跑用你的 Chrome）
```

## 使用（4 步）

1. **双击 `启动Chrome-调试模式.command`** —— 用调试模式重开你的 Chrome（保留登录态），在里面确认已登录 BOSS（zhipin.com）。
2. **`npm start`** —— 启动后台，浏览器自动打开控制台 `http://localhost:8848`。
3. 控制台「设置」里填 **API Key**（DeepSeek 的 `sk-...` 或 Claude 的 `sk-ant-...`，对应「评分模型」选项）、**作品集链接**；「规则」里设区域/薪资/关键词。
4. 点 **「连接 Chrome」** → 显示 ✅ → 点 **「流程一」**（打招呼）/**「流程二」**（监听发简历）。

> 建议第一次盯着看，确认动作正常；每日上限、HR 活跃时段、24h 持续都可在设置里调。

## 隐私 & 安全

- API Key、登录态、投递记录、统计**只存本地**（在系统应用数据目录），`.gitignore` 已排除，不会上传。
- 全程模拟真人节奏（随机延迟、拟人点击）。这是给本人求职用的工具，请控制频率，勿规模化滥用。

## 已知限制（实话实说）

- **不是 7×24 无人值守**：要你先用调试模式开着 Chrome；关掉 Chrome 就停。
- **薪资有时读不到**：BOSS 对薪资做了 DOM 混淆，少数卡片薪资程序读不到，会以 AI 评分 + 你看日志为准。
- BOSS 前端 class 经常变；选择器都在 `src/engine.js`，失效时改那里（`给ClaudeCode.md` 有真机实测的选择器对照表）。

## 文件

| 文件 | 作用 |
|------|------|
| `server.js` | 本地后台：HTTP + SSE，把界面和引擎接起来 |

| `src/index.html` | 网页控制台 UI（看板/规则/话术/日志/建议/设置）|

| `SPEC.md` / `给ClaudeCode.md` | 需求规格 / 真机实测选择器记录 |
