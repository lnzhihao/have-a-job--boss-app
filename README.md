# BOSS 找工作 Agent

BOSS 找工作 Agent 是一个面向个人求职场景的本地自动化工具。它连接用户自己已经登录的 Chrome 浏览器，在 BOSS 直聘页面里辅助完成岗位筛选、AI 匹配评分、半自动打招呼、HR 回复监听和作品集/简历发送提醒。

项目目标不是替代求职者，而是把重复、机械、容易漏掉的动作交给工具处理，让求职者把精力放在判断岗位、优化简历和正式沟通上。

## 页面入口

启动后会打开本地控制台：

```text
http://localhost:8848
```

主要页面文件：

| 页面 | 文件 | 说明 |
|---|---|---|
| 控制台页面 | `src/index.html` | 查看数据、设置规则、启动流程、查看日志和建议 |
| 本地后端 | `server.js` | 提供控制台页面、保存配置、推送实时日志、调用自动化引擎 |
| 自动化引擎 | `src/engine.js` | 连接 Chrome，执行岗位筛选、AI 评分、打招呼、监听回复 |

## 核心功能

- 连接真实 Chrome：使用用户本机已经登录的 Chrome，而不是新开一个容易被识别的自动化浏览器。
- 岗位筛选：支持按岗位关键词、城市区域、薪资范围、HR 活跃度等规则筛选。
- AI 匹配评分：读取岗位信息和个人背景，给岗位做匹配度判断。
- 流程一：在符合规则的岗位上执行半自动打招呼。
- 流程二：监听 HR 回复，发现回复后发送作品集链接，并尝试走 BOSS 原生简历发送流程。
- 实时看板：展示已打招呼、HR 回复、已发简历、已跳过等状态。
- 求职建议：根据简历、岗位方向和历史结果生成求职优化建议。

## 使用方式

安装依赖：

```bash
npm install
```

启动 Chrome 调试模式：

```bash
open -na "Google Chrome" --args --remote-debugging-port=9222
```

也可以直接双击项目里的：

```text
启动Chrome-调试模式.command
```

然后在这个 Chrome 里登录 BOSS 直聘：

```text
https://www.zhipin.com/web/geek/jobs
```

启动本地控制台：

```bash
npm start
```

打开：

```text
http://localhost:8848
```

在控制台里完成：

1. 填写 AI API Key。
2. 填写个人背景、目标岗位、作品集链接。
3. 设置城市、薪资、关键词、每日上限等规则。
4. 点击连接 Chrome。
5. 启动流程一或流程二。

## 项目亮点

- 针对真实求职流程设计，不是静态演示页面。
- 使用真实 Chrome 会话，保留用户已有登录态，减少被页面拦截的概率。
- 控制台、自动化引擎、实时日志拆分清楚，便于继续扩展。
- 支持两个独立流程：主动找岗位打招呼，以及 HR 回复后快速发送资料。
- 把 AI 用在匹配判断和求职建议上，而不是简单生成固定话术。

## 文件结构

| 文件 | 作用 |
|---|---|
| `server.js` | 本地服务入口，负责页面、配置、实时日志和流程控制 |

| `启动Chrome-调试模式.command` | 一键启动带调试端口的 Chrome |

| `SPEC.md` | 功能规格和产品设计说明 |
| `给ClaudeCode.md` | 选择器、实测流程和后续开发记录 |
| `reference/boss_agent.py` | 早期参考实现 |

## 隐私和安全

- API Key、登录态、简历路径、作品集链接、投递记录等配置保存在本机。
- 项目不会把用户简历或 BOSS 登录态上传到仓库。
- 本工具适合个人求职辅助使用，请控制频率，不建议用于批量滥用。

## 已知限制

- 必须先用调试模式打开 Chrome，并保持 BOSS 登录状态。
- BOSS 页面结构会变化，选择器偶尔需要维护。
- 简历发送依赖 BOSS 原生页面状态，部分情况下需要用户接手。
- AI 评分质量取决于岗位信息、个人背景和简历内容是否完整。

## 后续优化方向

- 增加岗位详情页截图或录屏，帮助作品集展示。
- 增加更清晰的“今日投递计划”和“岗位复盘”页面。
- 增加失败原因统计，例如薪资不匹配、区域不匹配、HR 不活跃。
- 增加多简历版本管理，根据岗位方向选择不同作品集或简历。
